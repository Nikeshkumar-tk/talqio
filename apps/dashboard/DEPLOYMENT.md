# TalqBot Dashboard - SST v3 Deployment Guide\n\n## 🚀 Quick Setup\n\n### Prerequisites\n- Node.js 18+ installed\n- AWS CLI configured with your credentials\n- An AWS account\n\n### 1. Install Dependencies\n```bash\nnpm install\n```\n\n### 2. Configure AWS Credentials\nMake sure your AWS credentials are configured:\n```bash\naws configure\n```\n\nOr set environment variables:\n```bash\nexport AWS_ACCESS_KEY_ID=your_access_key\nexport AWS_SECRET_ACCESS_KEY=your_secret_key\nexport AWS_DEFAULT_REGION=us-east-1\n```\n\n### 3. Initialize SST (First Time Only)\n```bash\nnpx sst@latest init\n```\nSelect:\n- ✅ Yes (Drop-in mode)\n- ✅ AWS\n\n### 4. Development\nStart local development with live reload:\n```bash\nnpm run dev\n```\nThis will:\n- Deploy your infrastructure to AWS\n- Start your Next.js app locally\n- Enable live Lambda functions\n- Create a local tunnel to your VPC\n\n### 5. Deploy to Production\n```bash\nnpm run deploy:prod\n```\n\n## 🛠️ Configuration\n\n### Custom Domain (Optional)\nTo use a custom domain, update `sst.config.ts`:\n1. Purchase domain in Route 53 or point nameservers to AWS\n2. Update the domain configuration in `sst.config.ts`\n3. Uncomment and configure the SSL certificate ARN\n\n### Environment Variables\nAdd environment variables in `sst.config.ts`:\n```typescript\nenvironment: {\n  NEXT_PUBLIC_API_URL: \"https://api.talqbot.com\",\n  DATABASE_URL: database.connectionString,\n}\n```\n\n### Add Database (Optional)\nUncomment the PostgreSQL configuration in `sst.config.ts` to add a managed database.\n\n## 📁 Project Structure\n```\n├── sst.config.ts          # SST configuration\n├── .sst/                  # SST generated files (gitignored)\n├── src/                   # Next.js app source\n├── package.json           # Updated with SST scripts\n└── README.md             # This file\n```\n\n## 🎯 Deployment Stages\n\n### Development\n```bash\nnpm run dev              # Personal dev stage\n```\n\n### Staging\n```bash\nsst deploy --stage staging\n```\n\n### Production\n```bash\nnpm run deploy:prod      # Production stage\n```\n\n## 🔧 Useful Commands\n\n```bash\n# Check app status\nsst status\n\n# View logs\nsst logs\n\n# Remove app (careful!)\nsst remove\n\n# Remove specific stage\nsst remove --stage staging\n\n# Update SST to latest\nnpm install sst@latest\n```\n\n## 📊 Monitoring\n\n1. **SST Console**: https://console.sst.dev\n   - View logs and metrics\n   - Set up auto-deployment\n   - Monitor issues\n\n2. **AWS Console**: Check CloudWatch, CloudFront, Lambda functions\n\n## 🔒 Security\n\n- All resources are deployed to your AWS account\n- Use IAM roles for least privilege access\n- Enable CloudTrail for audit logging\n- Configure WAF for additional security\n\n## 💰 Cost Optimization\n\n- **Free Tier**: Small apps can run nearly free\n- **Auto-scaling**: Pay only for what you use\n- **CloudFront**: Global CDN included\n- **Lambda**: Serverless pricing model\n\n## 🐛 Troubleshooting\n\n### Common Issues:\n\n1. **AWS Credentials**: Make sure they're configured correctly\n2. **Permissions**: Ensure your AWS user has appropriate permissions\n3. **Domain Issues**: Check DNS settings and certificate configuration\n4. **Build Errors**: Check Next.js build locally first\n\n### Getting Help:\n- SST Discord: https://discord.gg/sst\n- SST Documentation: https://sst.dev/docs\n- GitHub Issues: https://github.com/sst/sst/issues\n\n## 🚀 Next Steps\n\n1.
